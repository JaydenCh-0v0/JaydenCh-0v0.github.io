Slide 02 · Product Architecture

教學閉環：從對話到動態教材 的三層能力

第一層 · 即時交互 → 第二層 · 知識沉澱 → 第三層 · 動態教材
第一層 · 即時交互層

五種教育模式

Chat / Solve / Quiz / Research / Visualize 五模式共享同一學習上下文與引用歷史，學生無需在不同工具間切換，導師可全程追蹤學習軌跡。

  • 聊天（Chat）：引導式提問，釐清概念盲點
  • 解題（Solve）：逐步拆解思路，附引用出處
  • 測驗（Quiz）：自動出題與即時評分反饋
  • 研究（Research）：延伸閱讀與跨章節知識串聯
  • 視覺化（Visualize）：圖表、幾何與動態演示輔助理解
第二層 · 知識沉澱層

共同編劇 Markdown 筆記

雙欄分屏工作區讓師生同步編輯筆記，AI 可因應指令重寫、擴寫或精簡段落，將零散輔導對話轉化為結構化、可搜尋的個人知識庫。

  • 協作編輯：教師與學生同屏修訂，保留修訂歷史
  • AI 輔助改寫：一鍵調整語氣、難度或篇幅
  • 錯題沉澱：解題過程自動歸檔為個人錯題本
  • 教案複用：優質筆記可分享至班級或機構模板庫
第三層 · 動態教材層

圖書引擎 · 動態書籍

多智能體管道將靜態教材（PDF、Markdown、講義）自動編譯為可互動的「動態圖書」，內嵌測驗、閃卡與視覺化元件，實現名師經驗的規模化複製。

  • 自動編譯：上傳材料後一鍵生成章節化互動讀物
  • 內嵌測驗：每章附自動生成練習題與解析
  • 閃卡複習：關鍵概念抽取為間隔重複記憶卡
  • 幾何視覺化：整合 GeoGebra 等工具展示立體圖形

Slide 04 · Tech Stack

技術底座：Agent + RAG 如何支撐因材施教

白話解說

AI 私教並非「聊天機器人」這麼簡單。它背後靠兩招：一支 AI 助教團隊負責記住每位學生、分工教學；加上開卷查資料機制，確保每個答案都有課本依據，而非憑空捏造。

技術層 A · 教學代理

多智能體協同（Multi-Agent）

持久化 TutorBot 可衍生背景子智能體，自主規劃教學大綱、生成測驗與追蹤學習進度，實現「有記憶的導師」而非一次性問答。

  • 三層記憶架構保障跨節學習連貫性
  • 子智能體並行處理長週期教學任務
  • 教學過程可審計、可追溯
💬 解說

想像僱了一隊 AI 助教：前台導師跟學生聊天，背後還有專人負責出卷、排課、記錯題。重點是記得住這位學生上週問過什麼，不會每次見面都從零開始。

技術層 B · 知識引擎

檢索增強生成（RAG）

版本化文檔庫支援多源教材自動路由，解題與答疑均可引用原始出處，適合接入 DSE 真題、評卷標準等權威資料。

  • 多源文檔自動索引與路由
  • 安全引用與事實核查機制
  • 可對接校本教材與考評局資料
💬 解說

就像開卷考試——AI 回答前先翻課本、講義和 Past Paper 找依據，還能告訴你「答案出自第幾題、哪份 Marking Scheme」，避免胡亂編造。

Slide 07 · Production Pipeline

內容生產管線：從文案到成片 的四步閉環

模組 01

對口型（Lip-Sync）

整合 MuseTalk、LatentSync、Wav2Lip 等模型，將音頻與人物視頻精準對齊，實現自然的口型同步效果。

模組 02

文案提取（ASR）

基於 FunASR 語音識別引擎，支援帶時間戳的文字輸出與字幕一鍵導出，加速內容後期製作。

模組 03

高清化（Upscaling）

內建視頻超分辨率處理能力，提升數位人視頻的畫質與細節表現，滿足專業發佈需求。

模組 04

文本轉語音（TTS）

支援 CosyVoice、FishSpeech、GPT-SoVITS 等引擎，可基於少量參考音頻完成高保真聲音克隆。

Slide 09 · Tech Stack

技術底座：音容重構 與流式口型對齊

白話解說

數位人影片其實只做兩件事：一是讓嘴巴動作對準聲音（對口型）；二是用幾秒鐘錄音模仿你的聲線念任意台詞（聲音克隆）。其餘都是這兩步的自動化流水線。

技術層 A · 視覺合成

音視頻生成與口型對齊

採用擴散模型（Diffusion）或生成對抗網絡（GAN）對面部區域進行 Inpainting 重繪，配合音頻特徵矩陣在潛在空間完成流式口型同步。

  • 多模型可切換（MuseTalk / Wav2Lip 等）
  • 支援即時預覽與批量處理
  • 視頻超分提升輸出畫質
💬 解說

你提供一段靜態人像照片 + 一段錄音，AI 逐幀修改嘴部像素，讓嘴唇開合與聲音節拍一致——就像給照片裡的人「配嘴型」，而非從零生成整個人。

技術層 B · 靈魂重構

零樣本 / 少樣本聲音克隆

僅需 3–5 秒參考音頻即可還原語調與情感，為數位生命、虛擬主播及「賽博永生」項目的逝者音容重構提供底層能力。

  • 多 TTS 引擎可切換比較效果
  • ASR 提取 → TTS 克隆 → 對口型一條龍
  • 支援節點式工作流批量生產
💬 解說

錄3–5 秒你的聲音，AI 就能用「你的聲線」朗讀任意文字稿，無需逐句重錄。配合對口型，一段文字 → 你的聲音 → 你的臉在說話，全程自動完成。

核心商業想法（O2O）

多元引流場景
  • 毛孩經濟寵物照生成 Q 版立體公仔
  • 全家福紀念將平面合照立體化為客廳擺設
  • 情侶與婚慶客製化迷因互動模型，節慶送禮
  • 畢業里程碑畢業照融合學士熊 AI 打印

Slide 14 · Tech Stack

技術架構：AI 重建 與打印自動化

白話解說

從手機相片到手中公仔，技術上分兩步：AI 先「猜」出立體形狀（單圖 3D 重建），再由軟件「修」成能打印的文件（補洞、加厚、切片）。全程無需專業建模師。

技術模組 A · AI 重建

單圖 3D 重建（LRM）

基於 TripoSR 前饋 Transformer 架構，單張圖片一步到位預測 3D 結構，無需傳統 NeRF 數小時優化，滿足 C 端用戶「上傳即預覽」的即時體驗需求。

  • 0.5 秒級推理，適合 Web 端即時預覽
  • 6GB 顯存，消費級顯卡可本地部署
  • 支援紋理烘焙，輸出可打印網格
💬 解說

給 AI 看一張平面照片，它在半秒內腦補出沒拍到的背面和厚度，吐出一個可旋轉的 3D 模型——像資深雕塑師看一眼物件，就知道立體長什麼樣。

技術模組 B · 打印自動化

模型修復與硬件對接

調用 Blender API 自動網格閉合與加厚，確保模型符合打印規格；店面部署全彩多色打印機，降低消費者上手門檸，完成最後一公里實體交付。

  • 自動 Make Solid 修復可打印性
  • 標準 .obj 對接切片軟件
  • 全彩 AMS 系統支援多色情感公仔
💬 解說

3D 打印機只能印實心、封閉的物件。軟件自動把模型上的破洞補好、壁加厚，再轉成打印機語言——就像把一張紙折的模型，改成可以倒模出蠟像的模具。

精實驗證與 香港三階段落地

商業可行性

香港三階段驗證
