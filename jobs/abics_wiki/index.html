P104-ABICS

工業視覺 × 協作機器臂 × AI 檢測 — 端到端自動化螺栓分揀產線系統設計說明

System Design · P104-ABICS
01

系統總覽

專案目標、工位配置、整體架構與啟動流程

專案概覽

ABICS 將進料托盤上的螺栓經立體空間識別及抓取系統與識別引擎定位後，由 ABB GoFa 協作機器臂與 Hand-E 夾爪 完成拾取、清潔、卡盤對位與多視角物件表面與螺紋檢查，最後依檢測結果分揀至輸出托盤或廢料箱。 周邊軸系（卡盤、清潔、振動送料、工具切換等）由三菱 FX5 PLC 透過 G-code 執行器統一編排。

系統以 三個並行循環任務（Task A / B / C）協同運作，透過訊息佇列與互鎖機制串接整體 Job 生命週期。

典型 Job 資料流

進料托盤 ──振動A──► 識別托盤 ──Zivid+3D引擎──► 拾取佇列 │ Task A ◄──BoltComplete────┼────BoltReady──► Task B │ │ │ 清潔/水平/垂直/軸向工位 │ │ Task C ◄─InspectionStart─┘ InspectionComplete Lucid + 2D引擎/ZMQ │ 輸出托盤 / 廢料箱 ◄── Task B 分揀

啟動與初始化序列

系統初始化（AbicsFlowManager.StartService）

  1. 解析啟動參數（MongoDB 連線字串 / 系統名）。
  2. RbbMongoDb.ConnectAsync → 命令 LoadSystem 取得 AbicsSystemInfo。
  3. 遍歷 GetEquipInfos() 建立模組並 ConnectAllModules。
  4. UI 載入預設 Job、渲染輸出托盤，等待使用者啟動。

任務初始化（Job 開始前）

  1. 確認各模組無錯誤/警告，硬體回到準備姿態。
  2. 夾爪：未初始化則初始化；若持有螺栓則放回識別托盤。
  3. 機器人回到 standby。
  4. PLC 執行初始化 G-code：Z 升降至頂部、工具旋至物件表面與螺紋檢查系統、卡盤角度與夾爪復位。
  5. Task A/B/C 進入各自 Initializing → Idle，發送 TaskReadyMessage。

三任務 FSM 詳解

注意：Task A 為 Detection（進料識別），Task C 為 Inspection（品質檢查），兩者職責不同。

Task A輸入識別 · 振動進料 · 拾取佇列
→
Task B機器人 PnP · 清潔 · 對位 · 分揀
↔
Task C2D 拍攝 · 頭/側/軸檢查 · G-code 卡盤

Task A — 輸入識別 ▶ Demo · 3D 算法

  1. 等待 UI 啟動 Job，重置統計與輸出托盤 UI。
  2. Zivid 擷取識別托盤 RGB-D → 3D 引擎（SAM → PCA → ICP → 6D Model Sorting）→ 渲染結果。
  3. 無螺栓或靠邊過多 → 振動進料（最多 3 輪）→ 再掃描。
  4. 評分選最佳螺栓，計算 pick prepare / pick pose，推送拾取佇列。
  5. 向 Task B 發送「螺栓就緒」，等待「螺栓完成」。

Task B — 機器人 PnP ▶ Demo

  1. 自拾取佇列載入螺栓評估資料，執行拾取與夾持確認。
  2. 空氣清潔 → 頭部水平檢查工位（距離感測修正 head offset）→ 觸發 Task C 頭部頂視檢查。
  3. 垂直裝夾：卡盤夾六角頭，多輪 60° 翻面側視檢查（共 6 面）。
  4. 桿部軸向分段掃描（相機 Z 步進 + 卡盤 120° 分段）。
  5. 依結果放置輸出托盤或廢料，回報 Task A。

Task C — 檢查任務 ▶ Demo · YOLO

  1. 初始化：物件表面與螺紋檢查系統高度、工具旋至卡盤、卡盤與夾爪歸零。
  2. 依檢查類型：頭部前端 / 頭部側面 / 螺栓軸。
  3. Lucid 擷圖 → YOLO 檢測服務（ZMQ / Docker）。
  4. 結果寫入螺栓評估資料，回覆 Task B「檢查完成」。

PCA 對分割點雲做主成分分析，估計螺栓軸線方向與延伸（頭/尾/CG） 3d_dir_vec、3d_end、pca_sta [RMSE, MAE]
ICP 將實測點雲與標準螺栓 CAD/模板點雲迭代配準，精修 6D 位姿 transform (RrcPose)、icp_sta [FITNESS, RMSE]
6D Model Sorting 比對多規格 M×L 模型，依配準分數、穩定度、托盤位置評分排序 shaft_diameter、3d_len、stability_score → 拾取佇列

Task A 取評分最高者計算 pick prepare / pick pose；靠托盤邊緣的螺栓會降低優先或觸發振動進料。

2D 檢測算法（YOLO · BSv5 · BRCv3）

品質檢查由 Bolt_Inspection_System 內 YOLO（Ultralytics）實例分割/檢測模型驅動， 經 ZMQ 由 AbicsInspectClient 呼叫。幾何量測在 mask 基礎上結合 ISO 規格表驗證。

YOLO · BSv5 HexHeadDetector BoltSideDetector YOLO · BRCv3 BoltRustCrackDetector

模型 / 模組YOLO 目標類別檢查項
BSv5 — 頭部 (hexagon_transformation_detection) bolt_head_hexagon HEAD_STRUCTURE、六角內/外徑 (ISO)
BSv5 — 桿部 (thread_transformation_detection) bolt_side_thread, bolt_side_shank, bolt_side_hexaprism SIDE_STRUCTURE、節徑/小徑、PITCH；可疊加綠幕精修 mask
BRCv3 — 缺陷 (bolt_rust_crack_detection) RUST、CRACK 實例 腐蝕面積、裂紋 bbox + confidence

Task C 依視角觸發 inspection_type: head | side；UI/報告可重繪 inspections[] 標註框。

2D 檢測服務與 ZMQ 協定

Bolt_Inspection_System/ 提供可 Docker 部署的 ZMQ REP 服務，與 C# hk.asrc.abics.client.inspect 以 REQ/REP 通訊（預設 tcp://*:5657）。 YOLO 算法見 2D 檢測算法；標註圖見 2D 標註結果圖庫。

C# AbicsInspectClient Docker (bolt-inspection) Connect / CheckHeadFront / CheckVertical capture_server.py (ZMQ REP) │ Frame0: metadata JSON │ │ Frame1: image bytes ▼ └──────────────────────────────────► inspection_manager.py ├ hexagon_transformation (BSv5) ├ thread_transformation (BSv5) └ bolt_rust_crack (BRCv3) models/ · specifications/iso_specification.csv

ZMQ 請求（2 幀）

Frame 0: metadata JSON
Frame 1: BGR uint8 原始 bytes (width × height × channels)

// metadata 必填欄位
bolt_id, inspection_type ("head"|"side"), bolt_model, bolt_length,
pixel_to_mm_ratio, image_width, image_height, image_channels, image_dtype

預設位址 tcp://127.0.0.1:5657；客戶端 LinkInfo 範例：-server tcp://host:5657 -iso_csv ./iso_specification.csv -tolerance 0.1

回應 status

  • pass — 所有 inspection_result 為 NORMAL
  • fail — 任一 DEFECTED（含 RUST/CRACK）
  • error — 模型或管線異常

檢測項目（inspections[]）

inspection_type適用視角說明
HEAD_STRUCTUREheadYOLO 結構 bolt_head_hexagon
SIDE_STRUCTUREsidethread / shank / hexaprism
HEX_OUT_DIAMETER / HEX_IN_DIAMETERhead對 ISO 外徑/內徑
THREAD_PITCH_DIAMETER / THREAD_MINOR_DIAMETERside螺紋節徑、小徑
PITCHside螺距段量測
RUST / CRACKbothBRCv3 缺陷檢測

Python 管線（InspectionManager）

  • HexHeadDetector (BSv5) — 頭部幾何與六角量測
  • BoltSideDetector (BSv5) — 桿部結構與螺紋量測
  • BoltRustCrackDetector (BRCv3) — 鏽蝕與裂紋
  • ISOSpecLoader — specifications/iso_specification.csv

模型與 pixel_to_mm_ratio 見 functions/_config.yaml；執行檔 capture_server.py，v2 見 capture_server_v2.py / inspection_manager_v2.py。

docker build -t bolt-inspection:v1.0 .
docker run -d --gpus all -p 5657:5657 --name bolt_inspection_server bolt-inspection:v1.0
